What is Oracle wrapped PL/SQL?

Oracle wrapped PL/SQL is obfuscated source code. Developers and vendors use Oracle wrapping to hide readable implementation details while keeping packages, package bodies, functions, procedures, triggers, and type bodies executable inside the database.

A wrapped object usually starts with a statement such as CREATE OR REPLACE PACKAGE BODY ... wrapped, followed by version markers and encoded text. Oracle can compile and execute this output, but humans cannot easily read it without decoding.

When should you unwrap PL/SQL?

Only unwrap PL/SQL that you own, have permission to inspect, or are legally allowed to analyze. Do not unwrap third-party proprietary code without authorization.

  • Recover source for an internal package when the original repository copy is missing.
  • Review legacy PL/SQL before an Oracle migration or application modernization.
  • Document business logic in packages, procedures, and functions that your team maintains.
  • Debug production issues when wrapped source blocks normal code review.
  • Format decoded output so it can be copied, compared, or saved as a .sql file.

How to use UnwrapPLSQL

  1. Paste a full wrapped Oracle object, including the object header and final slash.

  5. Copy the result or download it as a clean SQL file.

Common issues

Input is not detected as wrapped

Make sure the input includes the word wrapped and the encoded body generated by Oracle. Partial snippets may not decode correctly.

Output formatting is not identical to the original source

Wrapped PL/SQL does not preserve all original whitespace and comments. Formatting creates readable code, but it cannot perfectly reconstruct every original style choice.

The code belongs to a vendor

Check your license and get permission before analyzing vendor-owned code. Use this tool for legitimate recovery, maintenance, and review workflows.
